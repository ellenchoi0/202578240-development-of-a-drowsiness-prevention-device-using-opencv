# Development-of-a-Drowsiness-Prevention-Device-Using-OpenCV

라즈베리파이와 Python(OpenCV)을 활용하여 실시간으로 안면 및 안구를 감지하고, 운전자의 졸음 상태를 판단하여 하드웨어 경보를 울려주는 졸음방지 자동화 알림 시스템 실습 프로젝트입니다.

> 본 프로젝트는 'OpenCV(Haar Cascade)' 분류기를 통해 얼굴 내부의 관심 영역(ROI)을 추출하고 눈의 개수를 판별하며,
> 'gpiozero' 라이브러리를 제어하여 조건별 능동부저 경보음 출력 및 예외 처리를 통한 자원 회수 메커니즘을 다룹니다.

## 개요

라즈베리파이 환경에서 웹캠 카메라 영상 데이터를 입력받아 매 프레임마다 실시간으로 안면 상태를 감지합니다. 연산 효율성을 최적화하기 위한 관심 영역(ROI) 기법을 학습하고, 탐지된 안구 수에 따른 하드웨어(능동부저) 제어 시스템을 구축합니다. 또한, 무한 루프 상태의 프로그램을 특정 키 입력('q')을 통해 안전하게 종료하는 메커니즘을 분석합니다.

## 핵심 특징

- **실시간 비전 제어**: 640x480 해상도 프레임 기반의 실시간 안면 및 안구 자동 추적
- **관심 영역(ROI) 최적화**: 1차 검출된 얼굴 좌표 행렬을 슬라이싱하여 눈 탐색 연산 범위를 대폭 축소
- **하드웨어 융합 경보**: 감지된 눈의 개수가 1개 이하일 때 GPIO 16번 능동부저 자동 제어
- **프로세스 안전 종료**: 사용자의 'q' 키 입력 인터럽트 감시 및 종료 시 카메라 세션 파괴 및 부저 소등

## 하드웨어 아키텍처

본 프로젝트는 [라즈베리파이의 BCM 핀 배열](https://www.raspberrypi.com/documentation/computers/raspberry-pi.html#gpio)을 기준으로 다음과 같이 설계되었습니다.

## 회로 구성 이론

- **Haar Cascade 분류 원리**: 사전에 학습된 XML 파일(frontalface_default, eye)을 로드하여 영상 속 특징점 기반 객체 탐지
- **관심 영역(ROI) 매트릭스 조작**: 명암(Gray) 변화를 추적하기 위해 흑백 이미지로 전처리한 후, 얼굴 좌표 gray[y:y+h, x:x+w] 영역만 분리하여 연산 병목 현상 방지

## 실행 및 설치

### 1. 의존성 설치

본 프로젝트 구동을 위해 가상환경 내에 OpenCV와 시스템 입출력 제어를 위한 gpiozero 패키지가 필요합니다.

### 2. 프로그 실행

터미널에서 소스코드가 있는 디렉토리로 이동 후 목적에 맞는 파이썬 스크립트를 구동합니다.
- 부저 연동 없이 비전 인식 단독 테스트 시: python3 main34.py
- 능동부저 경보 시스템 통합 구동 시: python3 main34-1.py

### 3. 종료(Interrupt)
- 실행 중 윈도우 창이나 터미널 환경에서 키보드 'q' 키를 입력하면 while 루프를 빠져나오며 안전하게 종료됩니다.
- 종료 직후 cv2.destroyAllWindows()와 buzzerPin.off() 로직이 수행되어 화면 인스턴스를 파괴하고 부저 잔류 전원을 완벽히 차단합니다.

## 주요 이슈 해결
- 연산 자원 한계와 ROI 적용: 제한된 성능의 라즈베리파이 보드에서 화면 전체를 탐색할 경우 프레임 드랍이 발생했으나, 얼굴 내부에만 눈 탐지기(eye_cascade) 범위를 한정 짓는 ROI 슬라이싱을 통해 버벅임 없는 실시간 연산 가동률을 확보함.
- 하드웨어-소프트웨어 동기화 및 예외 처리: 코드 상단에 정의하는 Buzzer(16)가 단순 변수가 아닌 실제 전선이 결선된 주소임을 인지하고 결선 상태를 디버깅함. 특히 예외적으로 프로그램 강제 종료 시 부저가 계속 울리는 오작동을 막기 위해 프로세스 종료 직전 소등 로직을 완벽히 매칭함.
- 환경 및 조도에 따른 비전 인식률의 한계: Haar Cascade 방식은 고개를 크게 돌리거나 조명이 어두우면 안면 감지 알고리즘 자체가 해제되어 눈을 추적할 수 없는 한계를 식별함. 향후 시스템 고도화를 위해 명암 변화에 강인한 딥러닝 기반 인공지능 Landmark 모델이나 적외선(IR) 카메라 하드웨어의 도입 필요성을 도출함.
